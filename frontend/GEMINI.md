# Frontend Guidelines

## Project Structure
- Next.js App Router lives in `frontend/app/`.
- Shared UI in `frontend/components/`.
- Hooks in `frontend/hooks/`.
- State stores in `frontend/stores/`.
- API and utilities in `frontend/lib/`.

## Development Commands
- Install: `cd frontend && npm install`
- Run (dev): `cd frontend && npm run dev`
- Tests: `cd frontend && npm test`

## Coding Style
- ESLint with Next.js and import ordering.
- Prettier with Tailwind plugin.
- Use `PascalCase` for components and `camelCase` for hooks/variables.
